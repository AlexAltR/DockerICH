# Останавливает и удаляет все контейнеры, сети, тома и образы, созданные после команды docker-compose up
docker-compose down

docker-compose up --build
# docker-compose — это инструмент для управления многоконтейнерными приложениями Docker с помощью конфигурационного файла docker-compose.yml. Он упрощает работу с контейнерами, их сборку, запуск, а также управление сетями и томами.

# up — эта команда запускает контейнеры, указанные в файле docker-compose.yml, и автоматически создает все необходимые контейнеры, сети и тома, если они еще не существуют. Если контейнеры уже существуют, то команда запустит их. Она также следит за логами контейнеров в реальном времени.

# --build — флаг, который заставляет Docker Compose пересобирать контейнеры перед запуском. Это полезно, когда вы вносили изменения в Dockerfile или в контекст (например, добавили новые зависимости в проект или изменили код). Это гарантирует, что контейнеры будут собраны заново с учетом всех изменений.
